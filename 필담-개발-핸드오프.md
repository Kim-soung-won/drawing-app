# 필담(Pildam) — 개발 핸드오프

> 디자인 시안(`필담.dc.html`)을 기존 **Pub Diary** (React Native + Expo + Skia) 코드에 적용하기 위한 명세서.
> 시안의 모든 수치·색은 기존 `src/constants/colors.ts` 토큰과 1:1로 맞춰져 있다.
> 이 문서는 `docs/` 에 두고 작업한다.

---

## 0. 한눈에 보기

| 항목 | 내용 |
|------|------|
| 테마 | **라이트 모드** (크림 종이 + 브라운/세이지 액센트) |
| 방향 | **가로/세로 가변** — `useWindowDimensions()` 의 `width > height` 로 판정 |
| 모드 | **필사**(손글씨 중심) · **그리기**(도형·다이어그램) 2종 |
| 핵심 화면 | 홈(라이브러리) · 필사 에디터 · 그리기 에디터 |
| 공통 셸 | 좌측 레일(74px 고정) + 보조 사이드바(가로만) + 메인 |

기존 앱과의 관계:
- **필사 모드 = 리스킨 + 레이아웃 재배치** (캔버스/저장/undo·redo/멀티페이지/S펜 전부 재사용)
- **그리기 모드 = 신규 기능** (도형 객체 모델·store·캔버스 신규 — 별도 Phase)

### 1.3 형태(Shape)

- 레일 아이콘 버튼 `borderRadius: 14`, 48×48
- 카드/썸네일 `borderRadius: 10`, **aspect-ratio 3:4**

## 2. 레이아웃 시스템 (반응형)

```
┌──────┬───────────┬──────────────────────────┐
│ RAIL │  SIDEBAR  │          MAIN            │
│ 74px │  212~226  │          flex:1          │
│ 고정 │  (가로만) │   topbar + content       │
└──────┴───────────┴──────────────────────────┘
```

### 판정 & 동작

```ts
const { width, height } = useWindowDimensions();
const isPortrait = height >= width;
```

| 요소 | 가로(landscape) | 세로(portrait) |
|------|-----------------|----------------|
| 레일(74px) | 항상 표시 | 항상 표시 |
| 보조 사이드바 | 표시 (212~226px) | **숨김** → topbar 햄버거로 Drawer 오픈 |
| 홈 카드 그리드 | **4열** | **2열** |
| 필사 종이 폭 | 560px 고정, max 88% | 화면폭 - 여백 (넓게) |
| topbar 햄버거 | 없음 | 좌측에 표시 |

> 세로 사이드바는 RN `Modal`/`Drawer`(예: `react-native-drawer-layout` 또는 자체 슬라이드 패널)로 오버레이.
> 시안에서는 `sideDisplay: none` + 햄버거 아이콘으로 동작을 표현.

**Topbar**:

- 우측: **템플릿 칩**(괘선·모눈·도트·무지) — 활성 `bg #8B7355 color #FFF8F0`

- **방향 토글**(가로/세로 세그먼트)

**캔버스(종이)** — 기존 `DrawingCanvas` + `PageBackground` 재사용:

### 3.4 그리기 에디터 — `app/draw.tsx` (신규)

기존 앱에 **없는 신규 기능**. 자유곡선(`Stroke`)이 아닌 **도형 객체** 중심.

**툴 사이드바** (신규):
- "도형": 사각형/원/삼각형/별/화살표/선/곡선/텍스트 (4열 그리드)
- "아이콘 라이브러리": 4열 그리드 (집/하트/시계/보내기/별/메모/카드/말풍선 등)
- "스티커": 🌿 ✦ ❀ ☾

**캔버스**:
- 무한 캔버스 느낌 — 도트 배경 `radial-gradient` 22px, `bg #FBF3E8`
- **Excalidraw 스타일 손그림 렌더링**: 직선이 아닌 떨림 있는 path (roughjs 알고리즘 / Skia path 변형)

**플로팅 툴바**:
- 선택 / 손그림 / 사각형 / 원 / 화살표 / 텍스트
- 색 4 (그리기 팔레트) + **거칠기(sloppiness) 3단** — Excalidraw 시그니처

**신규 데이터 모델** (`src/types/draw.ts`):
```ts
type ShapeKind = 'rect' | 'ellipse' | 'triangle' | 'arrow' | 'line' | 'freedraw' | 'text';
interface Shape {
  id: string;
  kind: ShapeKind;
  x: number; y: number; w: number; h: number;
  points?: Point[];          // freedraw/line
  text?: string;
  stroke: string; fill?: string;
  strokeWidth: number;
  roughness: 0 | 1 | 2;       // 거칠기
  rotation?: number;
}
interface DrawingDoc { version: 1; shapes: Shape[]; }
```
- 신규 `drawStore.ts` (선택/이동/리사이즈/undo·redo)
- 신규 `DrawCanvas.tsx` (Skia + Gesture: tap=선택, pan=이동/그리기, pinch=줌)

> **추천 라이브러리**: 손그림 path 는 `roughjs` 의 path 생성 알고리즘을 Skia `Path` 로 포팅하거나, 포인트에 노이즈를 더한 quadratic bezier 로 근사.

## 5. 구현 순서 (제안 Phase)

1. **토큰·폰트** — `colors.ts` 에 `SHELL` 추가, `expo-font` 로 4종 로드, 반응형 훅(`useOrientation`).

3. **홈 리스킨** — `CollectionSidebar` + 그리드(`numColumns` 4/2) + `NoteCard`.
4. **필사 리스킨** — `PageSidebar` + `TemplateChips` + `Toolbar` 플로팅화. (캔버스 로직 무손실)
5. **그리기 모드(신규)** — 데이터 모델 → `DrawCanvas` 기본 도형 → 손그림 렌더 → 선택/이동 → 아이콘·스티커.

> Phase 1–4 는 기존 자산 재사용이라 비교적 빠름. **Phase 5 가 실제 신규 개발 비중의 대부분**이므로 별도 스프린트로 분리 권장.
